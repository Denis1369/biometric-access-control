# Biometric Access Control

Система биометрического контроля доступа в реальном времени с распознаванием лиц, управлением сотрудниками и камерами, аналитикой и анализом загруженного видео.

## О проекте

**Biometric Access Control** — это прикладная информационная система для проходной/КПП. Проект объединяет backend на FastAPI, frontend на Vue, базу данных MySQL и модуль компьютерного зрения для работы с видеопотоками и фото.

Основной сценарий использования:
- подключение IP-камер;
- распознавание сотрудников по лицу;
- принятие решения о доступе;
- хранение сотрудников, гостей, камер и структуры объекта;
- просмотр журналов и аналитики;
- анализ загруженного видео для демонстрации и проверки сценариев.

## Основные возможности

- Аутентификация и авторизация пользователей.
- Ролевая модель доступа.
- Управление сотрудниками и их фотообразцами.
- Управление гостями.
- Управление камерами, зданиями, этажами и размещением камер на плане.
- Получение видеопотока и snapshot с камер.
- Аналитика по событиям доступа.
- Анализ загруженного видео с сохранением найденных событий.
- WebSocket-трансляция для просмотра потока с камер.

## Архитектура

Проект разделён на два основных приложения:

- `backend/` — API, бизнес-логика, работа с БД, обработка видео, распознавание и аналитика.
- `frontend/` — пользовательский интерфейс оператора и администратора.

### Основные backend-модули

- `auth` — вход в систему и работа с токеном.
- `employees` — сотрудники.
- `guests` — гости.
- `cameras` — камеры и их конфигурация.
- `buildings` / `floors` / `departments` — структура объекта.
- `analytics` — отчёты и сводные данные.
- `video_analysis` — анализ загруженного видео.
- `websockets` — потоковая передача видео на клиент.

## Роли доступа

Модель пользователей поддерживает следующие роли:

## Быстрый старт

### 1. Клонирование репозитория

```bash
git clone https://github.com/Denis1369/biometric-access-control.git
cd biometric-access-control
```

### 2. Настройка backend

Создай файл `.env` в каталоге `backend/`:

```env
DB_USER=root
DB_PASSWORD=root
DB_HOST=localhost
DB_PORT=3306
DB_NAME=scud_db

JWT_SECRET_KEY=change_me_to_a_long_random_secret
ACCESS_TOKEN_EXPIRE_MINUTES=120
```

Установи зависимости:

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
# Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

Запусти backend:

```bash
uvicorn app.main:app --reload
```

Backend будет доступен по адресу:

```text
http://localhost:8000
```

Swagger UI:

```text
http://localhost:8000/docs
```

### 3. Настройка frontend

Установи зависимости и запусти клиент:

```bash
cd frontend
npm install
npm run dev
```

Frontend будет доступен по адресу:

```text
http://localhost:5173
```

## Аутентификация

Проект использует JWT-аутентификацию.

- API-запросы отправляют токен в заголовке `Authorization: Bearer <token>`.
- Для WebSocket и некоторых media-url токен может передаваться через query parameter `token`.

## Анализ загруженного видео

В проекте предусмотрен отдельный модуль анализа загруженного видео. Это удобно для:

- демонстрации системы на защите;
- повторяемых сценариев тестирования;
- разбора коротких роликов без зависимости от live-камеры.

Результат анализа включает:
- статус задачи;
- количество обработанных кадров;
- количество разрешённых и запрещённых событий;
- список событий распознавания;
- превью найденных кадров.
