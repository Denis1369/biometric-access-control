# Ручная графовая разметка маршрутов

## 1. Назначение модуля

Модуль ручной графовой разметки нужен для описания доступных путей движения на плане этажа. Оператор открывает изображение плана, ставит технические точки маршрута и соединяет их линиями. В результате получается граф, по которому система может строить кратчайший путь между двумя точками.

Вершины графа не являются камерами, входами, кабинетами или другими объектами здания. Это только координаты на плане, которые помогают описать геометрию проходов.

## 2. Почему ручная разметка, а не нейросетевая

Для дипломного проекта ручная разметка надежнее и быстрее внедряется, чем автоматическая сегментация плана нейросетью. Планы зданий бывают разного качества: сканы, фотографии, эвакуационные схемы, дизайнерские изображения, планы с подписями и условными обозначениями. Нейросетевая разметка стен, дверей и коридоров потребовала бы отдельной модели, датасета, валидации и исправления ошибок распознавания.

Ручной граф проще контролировать: оператор сам задает только те проходы, по которым реально можно строить маршрут. Это снижает риск неверного пути через стену, закрытую дверь или служебную зону.

## 3. Как оператор ставит точки

Оператор выбирает здание и этаж с загруженным планом. В режиме разметки клик левой кнопкой мыши по пустому месту плана создает новую точку маршрута.

Точка хранит только координаты `x` и `y` в пикселях относительно оригинального изображения плана. Названия и типа у точки нет.

## 4. Как оператор соединяет точки линиями

Чтобы создать линию, оператор зажимает левую кнопку мыши на одной точке и тянет курсор к другой точке. Во время движения отображается временная пунктирная линия. Если отпустить кнопку над другой точкой, создается ребро графа. Если отпустить кнопку в пустом месте плана, система автоматически создаст там новую точку и сразу соединит ее с исходной точкой.

Если оператор кликает по уже существующей линии, система вставляет новую точку в эту линию: старое ребро удаляется, а вместо него создаются два ребра через новую точку. Если оператор тянет линию от точки и отпускает ее на существующей линии, линия также делится, а новая точка дополнительно соединяется с точкой, от которой начиналось протягивание.

Если связь между этими точками уже существует, дубль не создается. Правая кнопка мыши по точке удаляет точку и все связанные линии. Правая кнопка мыши по линии удаляет только эту линию.

## 5. Как система хранит граф

Граф хранится в двух таблицах:

`route_nodes` хранит точки маршрута:

```json
{
  "id": 1,
  "floor_id": 1,
  "x": 120,
  "y": 430
}
```

`route_edges` хранит линии между точками:

```json
{
  "id": 1,
  "floor_id": 1,
  "from_node_id": 1,
  "to_node_id": 2,
  "weight": 155.4,
  "is_bidirectional": true
}
```

Вес линии считается автоматически как евклидово расстояние между двумя точками:

```text
sqrt((x2 - x1)^2 + (y2 - y1)^2)
```

## 6. Как строится маршрут

Backend строит маршрут вручную алгоритмом Дейкстры без внешних библиотек. Узлы графа берутся из `RouteNode`, ребра из `RouteEdge`.

Если `is_bidirectional = true`, движение разрешено в обе стороны. Если `is_bidirectional = false`, движение разрешено только от `from_node_id` к `to_node_id`.

Если путь между выбранными точками существует, backend возвращает последовательность точек, ребра маршрута и общую длину. Если пути нет, API возвращает ошибку `404` с сообщением, что маршрут между выбранными точками не найден.

## 7. Применение в системе биометрического контроля доступа

В системе биометрического контроля доступа граф маршрутов используется как навигационная основа поверх плана здания. Камеры размещаются отдельно существующим функционалом, а граф описывает возможные проходы по этажу.

Когда система фиксирует гостя под разными камерами, эти события можно связать с ближайшими точками графа или зонами камер и построить понятную траекторию движения по плану. Такой подход отделяет видеонаблюдение и биометрию от геометрии здания: камеры отвечают за наблюдение, а ручной граф отвечает за корректные маршруты внутри помещения.
