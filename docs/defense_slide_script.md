# План презентации и текст доклада

## Как пользоваться этим планом

План рассчитан на доклад 7-10 минут. Оптимально сделать 13 слайдов: так рассказ остается цельным, но не превращается в чтение пояснительной записки.

На слайды выноси только короткие тезисы. Основные объяснения лучше говорить голосом. Комиссия должна видеть не «простыню текста», а логику: проблема → решение → архитектура → ключевой алгоритм → интерфейс → результат.

## Общая линия рассказа

Главная мысль доклада:

> Разработана система биометрического контроля доступа, которая регистрирует сотрудников и гостей, фиксирует события прохода с камер, хранит журналы и позволяет построить вероятный маршрут гостя по плану здания на основе событий камер, зон видимости и графа маршрутов.

## Слайд 8. Биометрическое распознавание

### Текст на слайде

**Два уровня идентификации**

**1. Распознавание лица**

- Используется, когда лицо видно достаточно хорошо.
- Сравниваются face embedding.

**2. Person Re-ID**

- Используется, когда лицо не видно.
- YOLO находит человека.
- OSNet извлекает признаки силуэта и одежды.

### Что показать

Схема:

Кадр → Лицо найдено → Face embedding → Гость  
Кадр → Лицо не найдено → YOLO person crop → OSNet body embedding → Гость

### Что говорить

В системе используется два подхода. Первый — распознавание лица. Если лицо видно, система вычисляет вектор признаков и сравнивает его с сохраненными векторами. Второй подход — Person Re-Identification. Он нужен, когда лицо плохо видно. YOLO находит человека в кадре, затем OSNet извлекает признаки силуэта и одежды, после чего система сравнивает их с сохраненным body embedding гостя.

## Слайд 10. Алгоритм вероятного маршрута

### Текст на слайде

**Построение маршрута**

1. Берутся события гостя по времени.
2. Для каждой камеры берется зона видимости.
3. Определяются участки графа, пересекающие зону камеры.
4. Между соседними событиями строится кратчайший путь.
5. Направление маршрута определяется по timestamp.

**Важно:** маршрут является вероятным.

### Что показать

Схема:

Camera02 → Camera03 → Camera05 → Camera06  
↓  
Зоны видимости  
↓  
Пересечение с графом  
↓  
Dijkstra  
↓  
Вероятный маршрут

### Что говорить

Камера в системе не считается точкой маршрута. Камера только дает событие: гость был замечен в зоне видимости в определенное время. Система берет последовательность таких событий, например Camera02, затем Camera03, затем Camera05 и Camera06. Для каждой камеры определяется участок графа маршрутов, который пересекает ее зону видимости. Затем между соседними участками строится кратчайший путь по алгоритму Дейкстры. Направление движения берется только из порядка событий по времени.

## Заготовки ответов на вопросы комиссии

### Почему маршрут называется вероятным?

Потому что камера фиксирует гостя внутри зоны видимости, но не определяет точную координату человека в каждый момент времени. Система строит наиболее вероятный путь по размеченному графу маршрутов на основе порядка событий камер.

### Почему не строить маршрут напрямую от камеры к камере?

Камера не является физической точкой движения человека. Она только наблюдает область. Поэтому маршрут строится не между камерами, а между участками графа, которые попали в зоны видимости этих камер.

### Зачем нужен Re-ID, если есть распознавание лица?

Распознавание лица работает хорошо, когда лицо видно. В реальных камерах лицо часто закрыто, находится под плохим углом или в плохом освещении. Re-ID дополняет распознавание лица и позволяет искать гостя по силуэту и одежде.

### Почему граф маршрутов размечается вручную?

Ручная разметка надежнее и контролируемее для практической системы. Автоматическое распознавание стен, дверей и коридоров требует отдельной обученной модели и стабильного качества планов. В проекте выбран подход, который легко проверить и использовать в демонстрационном сценарии.

### Где используется алгоритм Дейкстры?

Алгоритм Дейкстры используется для поиска кратчайшего пути по графу маршрутов между участками, связанными с последовательными событиями камер.

### Что можно улучшить дальше?

Можно добавить более точную калибровку камер, автоматическую разметку планов, хранение preview-кадров событий, настройку порогов Re-ID через интерфейс и расширенную аналитику по маршрутам.
